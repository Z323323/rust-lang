>The ability to run some code depending on **whether** a condition is `true` and the ability to run some code repeatedly **while** a condition is `true` are basic building blocks in most programming languages. The most common constructs that let you control the flow of execution of Rust code are `if` **expressions** and loops.

## `if` Expressions

>An `if` expression allows you to branch your code depending on conditions.

>All `if` expressions start with the keyword `if`, followed by a condition. In Code 3.5.1, the condition checks whether or not the variable `number` has a value less than `5`. We place the block of code to execute if the condition is `true` immediately after the condition inside curly brackets. Optionally, we can also include an `else` expression, to give the program an alternative block of code to execute should the condition evaluate to `false`. **If you don’t provide an `else` expression and the condition is `false`, the program will just skip the `if` block and move on to the next bit of code.** Try running Code 3.5.1; you should see E-Output 3.5.1.

<CodeGroup>
```rust Code 3.5.1
fn main() {
    let number = 7;

    if number < 5 {
        println!("condition was true");
    }

    println!("hehe");
}
```
```text E-Output 3.5.1

    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.07s
     Running `target/debug/branches`
hehe
```
</CodeGroup>

>It’s also worth noting that the `if` condition must evaluate to a `bool` expression. If the condition isn’t a `bool`, we’ll get an error. 

### Handling Multiple Conditions with `else if`

You can use multiple conditions by combining `if` and `else` in an `else if` expression. For example, when Code 3.5.2 executes, it checks each `if` expression in turn and executes the first body for which the condition evaluates to `true`. Note that even though `6` is divisible by `2`, we don’t see the output `number is divisible by 2` in E-Output 3.5.2, nor do we see `number is not divisible by 4, 3, or 2` text from the `else` block. That’s because **Rust only executes the block for the first `true` condition, and once it finds one, it doesn’t even check the rest**.

<CodeGroup>
```rust Code 3.5.2
fn main() {
    let number = 6;

    if number % 4 == 0 {
        println!("number is divisible by 4");
    } else if number % 3 == 0 {
        println!("number is divisible by 3");
    } else if number % 2 == 0 {
        println!("number is divisible by 2");
    } else {
        println!("number is not divisible by 4, 3, or 2");
    }
}
```
```text E-Output 3.5.2
   Compiling branches v0.1.0 (file:///projects/branches)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.31s
     Running `target/debug/branches`
number is divisible by 3
```
</CodeGroup>

### Using `if` in a `let` Statement

>Because `if` is an **expression**, we can use it on the right side of a `let` statement to assign the outcome to a variable, as in Code 3.5.3.

<CodeGroup>
```rust Code 3.5.3
fn main() {
    let condition = false;
    let number = if condition {
        5
    } else {
        unsafe { *("hehe" as *const str as *const u32) }
    };

    println!("The value of number is: {number}");
}
```
```text E-Output 3.5.3
   Compiling branches v0.1.0 (/var/home/z561/rust_projects/branches)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.15s
     Running `target/debug/branches`
The value of number is: 1701340520
```
</CodeGroup>

>**Remember that blocks of code evaluate to the last expression in them**, and numbers by themselves are also expressions. In this case, the value of the whole `if` expression depends on which block of code executes. This means the values that have the potential to be results from each arm of the `if` **must be the same type**. If the types are mismatched, as in Code 3.5.4, we’ll get an error (C-Output 3.5.4).

<CodeGroup>
```rust Code 3.5.4
fn main() {
    let condition = false;
    let number = if condition { 5 } else { "hehe" };

    println!("The value of number is: {number}");
}
```
```text C-Output 3.5.4

  |
3 |     let number = if condition { 5 } else { "hehe" };
  |                                 -          ^^^^^^ expected integer, found `&str`
  |                                 |
  |                                 expected because of this

For more information about this error, try `rustc --explain E0308`.
error: could not compile `branches` (bin "branches") due to 1 previous error
```
</CodeGroup>

>**The expression in the `if` block evaluates to an integer, and the expression in the `else` block evaluates to a string.** This won’t work, because variables must have a single type, and Rust needs to know definitively at compile time what type the `number` variable is. Knowing the type of `number` lets the compiler verify the type is valid everywhere we use `number`. **Rust wouldn’t be able to do that if the type of number was only determined at runtime**; the compiler would be more complex and would make fewer guarantees about the code if it had to keep track of multiple hypothetical types for any variable.

## Repetition with Loops

>It’s often useful to execute a block of code more than once. For this task, Rust provides several _loops_, which will run through the code inside the loop body to the end and then start immediately back at the beginning.

>Rust has three kinds of loops: `loop`, `while`, and `for`. Let’s try each one.

### Repeating Code with `loop`

>The `loop` keyword tells Rust to execute a block of code over and over again either forever or until you explicitly tell it to stop.

### Returning Values from Loops

>One of the uses of a `loop` is to retry an operation you know might fail, such as checking whether a thread has completed its job. You might also need to pass the result of that operation out of the loop to the rest of your code. To do this, you can add the value you want returned after the `break` expression you use to stop the loop; that value will be returned out of the loop so that you can use it, as shown in Code 3.5.6:

```rust Code 3.5.6
fn main() {
    let mut counter = 0;

    let result = loop {
        counter += 1;

        if counter == 10 {
            break counter * 2; // Returns '20'.
        }
    };

    println!("The result is {result}"); // 'The result is 20'
}
```

>You can also `return` from inside a `loop`. While `break` only exits the current loop, **`return` always exits the current function**.

### Disambiguating with Loop Labels

>**If you have loops within loops, `break` and `continue` apply to the innermost loop at that point. You can optionally specify a _loop label_ on a loop that you can then use with `break` or `continue` to specify that those keywords apply to the labeled loop instead of the innermost loop. Loop labels must begin with a single quote.** Here’s an example with two nested loops:

<CodeGroup>
```rust Code 3.5.7
fn main() {
    let mut count = 0;
    'counting_up: loop {
        println!("count = {count}");
        let mut remaining = 10;

        loop {
            println!("remaining = {remaining}");
            if remaining == 9 {
                break;
            }
            if count == 2 {
                break 'counting_up;
            }
            remaining -= 1;
        }

        count += 1;
    }
    println!("End count = {count}");
}
```
```text E-Output 3.5.7
   Compiling loops v0.1.0 (/var/home/z561/rust_projects/loops)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.08s
     Running `target/debug/loops`

remaining = 10
remaining = 9
count = 1
remaining = 10
remaining = 9
count = 2
remaining = 10
End count = 2
```
</CodeGroup>

The actual execution performs the following steps:

1. The outer loop starts, setting `remaining = 10`, and immediately triggers the inner loop.
2. The inner loop decreases the `remaining` variable to `9`, restarts, and then hits a standard `break` (because `remaining == 9`), exiting only the inner loop.
3. The outer loop then increments the `count` (`count += 1`), bringing the `count` to `1`, and restarts from the beginning.
4. On the next iteration, '1.', '2.', and '3.' are repeated so that the outer loop increments the `count` to 2.
5. '1.' is repeated, but not '2.'. When the execution reaches the specific condition `if count == 2`, `break 'counting_up` is triggered. This explicitly breaks the outer loop from within the inner one, ending the execution and printing `End count = 2`.

### Returning Explicitly from Bare Blocks

Recall from section 3.2 where I hinted that values could be returned from a bare `{}` block explicitly. The approach is strictly related to the constructs we have just examined, with the sole exception that the block is not defined as a `loop`. This is demonstrated in Code 3.5.8, with its effectiveness proven by E-Output 3.5.8.

<CodeGroup>
```rust Code 3.5.8
fn main() {
    let result = 'my_block: {
        println!("We are inside the block, doing some calculations...");
        
        let a = 10;
        let b = 32;

        // Explicitly returning the value from the block unconditionally.
        break 'my_block a + b; 

        // Any code placed after the break will be completely ignored.
        // In fact, the Rust compiler will give you an "unreachable statement" warning!
        // println!("This will never be printed.");
    };

    println!("The result is: {}", result);
}
```
```text E-Output 3.5.8
   Compiling loops v0.1.0 (/var/home/z561/rust_projects/loops)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.07s
     Running `target/debug/loops`
We are inside the block, doing some calculations...
The result is: 42
```
</CodeGroup>

You might be wondering if you can simply use a `break` expression without a label. The short answer is: **no**. **If you are inside a bare `{}` block (and not inside a `loop`, `while`, or `for` loop), an unlabeled `break` is not allowed.** In Rust, an unlabeled `break` (such as `break 42;`) will always try to target the innermost loop. If you attempt to use it to exit a bare block, the compiler will instantly throw `Error E0268` (`break` outside of a loop). Therefore, to explicitly break out of a bare block, the label is strictly mandatory.

### Streamlining Conditional Loops with `while`

>A program will often need to evaluate a condition within a loop. While the condition is `true`, the loop runs. When the condition ceases to be `true`, the program calls `break`, stopping the loop. It’s possible to implement behavior like this using a combination of `loop`, `if`, `else`, and `break`, however, this pattern is so common that Rust has a built-in language construct for it, called a `while` loop. In Code 3.5.9, we use `while` to loop the program three times, counting down each time, and then, after the loop, to print a message and exit.

```rust Code 3.5.9
fn main() {
    let mut number = 3;

    while number != 0 {
        println!("{number}!"); // '3', '2', '1'

        number -= 1;
    }

    println!("LIFTOFF!!!");
}
```

>This construct eliminates a lot of nesting that would be necessary if you used `loop`, `if`, `else`, and `break`, and it’s clearer. While a condition evaluates to `true`, the code runs; otherwise, it exits the loop.

### Looping Through a Collection with `for`

>You can choose to use the `while` construct to loop over the elements of a collection, such as an array. For example, the loop in Code 3.5.10 prints each element in the array `a`.

<CodeGroup>
```rust Code 3.5.10
fn main() {
    let a = [10, 20, 30, 40, 50];
    let mut index = 0;

    while index < 5 {
        println!("the value is: {}", a[index]);

        index += 1;
    }
}
```
```text E-Output 3.5.10
   Compiling loops v0.1.0 (/var/home/z561/rust_projects/loops)
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.06s
     Running `target/debug/loops`
the value is: 10
the value is: 20
the value is: 30
the value is: 40
the value is: 50
```
</CodeGroup>

>However, Code 3.5.10 approach is error-prone; we could cause the program to panic if the index value or test condition is incorrect. For example, if you changed the definition of the `a` array to have four elements but forgot to update the condition to `while index < 4`, the code would panic. **It’s also slow, because the compiler adds runtime code to perform the conditional check of whether the index is within the bounds of the array on every iteration through the loop.**

>As a more concise alternative, you can use a `for` loop and execute some code for each item in a collection. A `for` loop looks like Code 3.5.11.

```rust Code 3.5.11
fn main() {
    let a = [10, 20, 30, 40, 50];

    for element in a {
        println!("the value is: {element}");
    }
}
```

>When we run Code 3.5.11, we’ll see the same output as in E-Output 3.5.10. More importantly, we’ve now increased the safety of the code and eliminated the chance of bugs that might result from going beyond the end of the array or not going far enough and missing some items. **Machine code generated from `for` loops can be more efficient as well because the index doesn’t need to be compared to the length of the array at every iteration.**

>The safety and conciseness of `for` loops make them the most commonly used loop construct in Rust. Even in situations in which you want to run some code a certain number of times, as in the countdown example that used a `while` loop in Code 3.5.9, **most Rustaceans would use a `for` loop**. **The way to do that would be to use a `Range`, provided by the standard library, which generates all numbers in sequence starting from one number and ending before another number.**

>Here’s what the countdown would look like using a `for` loop and another method we’ve not yet talked about, `rev`, to reverse the range:

```rust Code 3.5.12
fn main() {
    for number in (1..4).rev() { // 1 to 3 reversed.
        println!("{number}!"); // '3', '2', '1'
    }
    println!("LIFTOFF!!!");
}
```
