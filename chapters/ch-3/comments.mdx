Comments let you annotate source code for human readers without affecting how the program compiles or runs. Rust has three kinds of comments: regular line comments, outer documentation comments, and inner (crate-level) documentation comments. The latter two are especially powerful because the `rustdoc` tool reads them and generates HTML documentation automatically.

## Line Comments

The most common comment style starts with two forward slashes (`//`). Everything from `//` to the end of the line is ignored by the compiler.

```rust
fn main() {
    // This is a full-line comment — the compiler ignores it entirely.
    let x = 5; // Inline comment: x holds the value 5

    // You can stack multiple line comments
    // to write longer explanations across
    // several lines.
    println!("x = {x}");
}
```

Rust does not have a dedicated multi-line block comment syntax for regular comments, but `/* ... */` block comments are supported by the compiler even if they are rarely used in idiomatic Rust.

## Documentation Comments (`///`)

Outer documentation comments use three slashes (`///`) and must be placed immediately before the item they document—a function, struct, enum, module, or trait. They support Markdown formatting and are collected by `rustdoc` to build API documentation.

```rust
/// Adds two 32-bit integers together and returns their sum.
///
/// # Examples
///
/// ```
/// let result = my_crate::add(2, 3);
/// assert_eq!(result, 5);
/// ```
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

/// A point in two-dimensional space.
pub struct Point {
    /// The horizontal coordinate.
    pub x: f64,
    /// The vertical coordinate.
    pub y: f64,
}
```

The `# Examples` section is more than just documentation—`cargo test` will automatically compile and run the code inside `///` doc-comment code fences to verify they stay correct.

## Crate-Level Doc Comments (`//!`)

Inner documentation comments use `//!` and document the *item that contains them* rather than the item that follows. They are typically placed at the very top of `src/lib.rs` or `src/main.rs` to describe the crate or module as a whole.

```rust
//! # My Crate
//!
//! `my_crate` provides a small set of utility functions for doing
//! simple arithmetic operations.
//!
//! ## Quick Start
//!
//! Add this to your `Cargo.toml`:
//!
//! ```toml
//! [dependencies]
//! my_crate = "0.1"
//! ```

pub fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

Running `cargo doc --open` on any Rust project will generate and open the HTML documentation derived from all `///` and `//!` comments in your code.
