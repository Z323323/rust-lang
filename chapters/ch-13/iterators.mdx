Iterators are one of Rust's most expressive features. They provide a uniform, composable API for processing sequences of values â€” whether those values come from a `Vec`, a file, a network socket, or are generated on the fly. Crucially, Rust iterators are **lazy**: no computation happens until you actually consume the iterator, which means you can chain multiple operations without paying for intermediate allocations.

## The `Iterator` Trait

The core trait is defined in the standard library:

```rust
pub trait Iterator {
    type Item;
    fn next(&mut self) -> Option<Self::Item>;
    // ... dozens of default methods built on top of `next`
}
```

Every iterator must implement just one method: `next`. It returns `Some(item)` as long as there are items, and `None` when exhausted. All the rich adapter methods (`map`, `filter`, `zip`, `enumerate`, â€¦) are provided as default implementations in terms of `next`.

### Manual `next` Calls

You can call `next` directly, though you rarely need to:

```rust
fn main() {
    let v = vec![1, 2, 3];
    let mut iter = v.iter();

    println!("{:?}", iter.next()); // Some(1)
    println!("{:?}", iter.next()); // Some(2)
    println!("{:?}", iter.next()); // Some(3)
    println!("{:?}", iter.next()); // None
}
```

Note: iterators must be declared `mut` because `next` takes `&mut self`.

## Creating Iterators from Collections

Three standard methods on collections produce different iterator flavors:

| Method | Yields | Ownership |
|--------|--------|-----------|
| `.iter()` | `&T` | Borrows the collection |
| `.iter_mut()` | `&mut T` | Mutably borrows the collection |
| `.into_iter()` | `T` | Consumes the collection |

```rust
let v = vec![1, 2, 3];
let sum_refs: i32 = v.iter().sum();           // v still usable
let sum_owned: i32 = v.into_iter().sum();     // v moved, can't use after
```

## Consuming Adaptors

Methods that call `next` until the iterator is exhausted are called **consuming adaptors**:

### `sum`

```rust
let numbers = vec![1, 2, 3, 4, 5];
let total: i32 = numbers.iter().sum();
println!("Sum: {total}"); // Sum: 15
```

### `collect`

Drives the iterator to completion and stores results in a collection. The type annotation tells Rust what to build:

```rust
let doubled: Vec<i32> = vec![1, 2, 3]
    .iter()
    .map(|x| x * 2)
    .collect();
println!("{:?}", doubled); // [2, 4, 6]
```

### `count`, `max`, `min`, `any`, `all`

```rust
let v = vec![3, 1, 4, 1, 5, 9];
println!("{}", v.iter().count());            // 6
println!("{:?}", v.iter().max());            // Some(9)
println!("{}", v.iter().any(|&x| x > 8));   // true
println!("{}", v.iter().all(|&x| x > 0));   // true
```

## Iterator Adaptors

Iterator adaptors transform one iterator into another. They are lazy â€” they do no work until consumed.

### `map`

Transforms each element:

```rust
let squares: Vec<i32> = (1..=5).map(|x| x * x).collect();
println!("{:?}", squares); // [1, 4, 9, 16, 25]
```

### `filter`

Keeps only elements for which the predicate returns `true`:

```rust
let evens: Vec<i32> = (1..=10).filter(|x| x % 2 == 0).collect();
println!("{:?}", evens); // [2, 4, 6, 8, 10]
```

### `enumerate`

Pairs each element with its index:

```rust
let words = vec!["hello", "world", "rust"];
for (i, word) in words.iter().enumerate() {
    println!("{i}: {word}");
}
// 0: hello
// 1: world
// 2: rust
```

### `zip`

Combines two iterators into an iterator of tuples:

```rust
let names = vec!["Alice", "Bob"];
let scores = vec![95, 87];
let paired: Vec<_> = names.iter().zip(scores.iter()).collect();
println!("{:?}", paired); // [("Alice", 95), ("Bob", 87)]
```

### `flat_map`

Maps then flattens one level:

```rust
let words = vec!["hello world", "foo bar"];
let chars: Vec<&str> = words.iter()
    .flat_map(|s| s.split_whitespace())
    .collect();
println!("{:?}", chars); // ["hello", "world", "foo", "bar"]
```

### `take` and `skip`

```rust
let first_three: Vec<i32> = (0..).take(3).collect();  // [0, 1, 2]
let skip_two: Vec<i32> = (0..5).skip(2).collect();    // [2, 3, 4]
```

## Chaining Adaptors

The real power comes from chaining:

```rust
let result: Vec<String> = (1..=10)
    .filter(|x| x % 2 != 0)   // keep odd numbers
    .map(|x| x * x)             // square them
    .map(|x| format!("{x}"))    // convert to string
    .collect();

println!("{:?}", result); // ["1", "9", "25", "49", "81"]
```

The compiler fuses all these operations into a single pass â€” no intermediate `Vec`s are allocated.

## Implementing Your Own Iterator

```rust
struct Counter {
    count: u32,
    max: u32,
}

impl Counter {
    fn new(max: u32) -> Counter {
        Counter { count: 0, max }
    }
}

impl Iterator for Counter {
    type Item = u32;

    fn next(&mut self) -> Option<Self::Item> {
        if self.count < self.max {
            self.count += 1;
            Some(self.count)
        } else {
            None
        }
    }
}

fn main() {
    let sum: u32 = Counter::new(5)
        .zip(Counter::new(5).skip(1))
        .map(|(a, b)| a * b)
        .filter(|x| x % 3 == 0)
        .sum();
    println!("{sum}"); // 6
}
```

By implementing only `next`, we get every adapter method for free.

## Summary

- The `Iterator` trait requires only `next`; all other methods are built on it.
- Iterators are lazy â€” chain as many adaptors as you like with no intermediate cost.
- **Consuming adaptors**: `sum`, `collect`, `count`, `any`, `all` â€” exhaust the iterator.
- **Iterator adaptors**: `map`, `filter`, `zip`, `enumerate`, `flat_map` â€” produce new iterators.
- Implement `Iterator` for your own types by defining `Item` and `next`.
