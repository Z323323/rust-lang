Enums (short for *enumerations*) let you define a type by listing its possible *variants*. Unlike enums in C or Java — which are little more than named integers — Rust enums can carry data of different shapes in each variant, making them a powerful tool for modelling domain concepts precisely.

## A Basic Enum

Here is a classic example: the directions of an IP address protocol version.

```rust
enum IpAddrKind {
    V4,
    V6,
}

fn route(kind: IpAddrKind) {
    match kind {
        IpAddrKind::V4 => println!("Routing IPv4"),
        IpAddrKind::V6 => println!("Routing IPv6"),
    }
}

fn main() {
    let four = IpAddrKind::V4;
    let six  = IpAddrKind::V6;
    route(four);
    route(six);
}
```

Variants are namespaced under the enum with `::`. Both `V4` and `V6` are of type `IpAddrKind`, so functions can accept either.

## Enums with Data

Rust's real power shows when you attach data directly to variants — each variant can hold a different type and amount of data:

```rust
enum IpAddr {
    V4(u8, u8, u8, u8),   // four octets
    V6(String),            // full string representation
}

fn main() {
    let home    = IpAddr::V4(127, 0, 0, 1);
    let loopback = IpAddr::V6(String::from("::1"));
}
```

This is more expressive than using a struct with a separate kind field, because the enum guarantees that a `V4` address always has four octets and a `V6` address always has a string.

### Enums with Struct-Like Variants

Variants can also have named fields, like a struct:

```rust
enum Message {
    Quit,                         // no data
    Move { x: i32, y: i32 },     // named fields
    Write(String),                // single String
    ChangeColor(i32, i32, i32),   // three i32 values
}
```

One enum type (`Message`) replaces four separate struct definitions. You can even implement methods on an enum with `impl`:

```rust
impl Message {
    fn call(&self) {
        match self {
            Message::Quit           => println!("Quit"),
            Message::Move { x, y } => println!("Move to ({}, {})", x, y),
            Message::Write(text)   => println!("Write: {}", text),
            Message::ChangeColor(r, g, b) => println!("Color: ({}, {}, {})", r, g, b),
        }
    }
}
```

## `Option<T>`: Rust's Null-Safe Type

Rust has no `null`. Instead, the standard library provides `Option<T>`, defined as:

```rust
enum Option<T> {
    None,
    Some(T),
}
```

`Option<T>` is so fundamental that it is in the prelude — you don't need to import it or prefix `Some`/`None` with `Option::`.

```rust
fn find_first_even(numbers: &[i32]) -> Option<i32> {
    for &n in numbers {
        if n % 2 == 0 {
            return Some(n);
        }
    }
    None
}

fn main() {
    let nums = vec![1, 3, 5, 4, 7];
    match find_first_even(&nums) {
        Some(n) => println!("First even: {}", n), // 4
        None    => println!("No even numbers"),
    }
}
```

### Why `Option<T>` Is Safer Than Null

In languages with null, *any* value of a reference type might be null, and forgetting to check causes a `NullPointerException` at runtime. In Rust, a value of type `i32` is *always* a valid integer — it can never be null. Only a value of type `Option<i32>` might be absent. The compiler forces you to handle the `None` case before you can use the inner value, turning a runtime error into a compile-time guarantee.

```rust
fn main() {
    let some_number: Option<i32> = Some(42);
    let absent: Option<i32> = None;

    // You cannot use an Option<i32> directly as an i32:
    // let total = some_number + 1; // ❌ compile error

    // You must extract the value first:
    if let Some(n) = some_number {
        println!("Got: {}", n + 1); // ✅
    }
}
```

### Useful `Option` Methods

`Option<T>` comes with many convenience methods:

```rust
fn main() {
    let opt: Option<i32> = Some(10);

    println!("{}", opt.unwrap());              // 10 (panics if None)
    println!("{}", opt.unwrap_or(0));          // 10 (returns 0 if None)
    println!("{}", opt.unwrap_or_else(|| 99)); // 10 (calls closure if None)
    println!("{}", opt.is_some());             // true
    println!("{}", opt.is_none());             // false

    let doubled = opt.map(|n| n * 2);
    println!("{:?}", doubled);                 // Some(20)

    let none: Option<i32> = None;
    println!("{}", none.unwrap_or(42));        // 42
}
```

## Summary

- Enums define a type with a fixed set of named variants.
- Each variant can carry data: unit, tuple-like, or struct-like.
- Rust enums are *algebraic data types* — far more expressive than C-style enums.
- `Option<T>` replaces null: `Some(T)` for a present value, `None` for absence.
- The compiler forces you to handle both `Some` and `None`, eliminating null-pointer bugs.
