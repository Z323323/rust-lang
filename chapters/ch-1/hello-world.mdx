In Rust we use `snake_case` to name files and directories as well.

<Steps>
  <Step title="Project Setup">
  Copy, paste, and run Command 1.2.1.
  
    ```bash Command 1.2.1
    cd ~ &&
    mkdir rust_projects &&
    cd rust_projects &&
    mkdir hello_world &&
    cd hello_world &&
    gedit hello_world.rs
    ```
  </Step>

  <Step title="Writing hello_world.rs">
  Add the following code to your _hello_world.rs_ file and make sure to save the file before closing `gedit`.
  
    ```rust ~/rust_projects/hello_world/hello_world.rs
    fn main() {
    println!("Hello, world!");
    }
    ```
  </Step>

  <Step title="Compiling and Running">
    <Note>
    Whenever an expected result is indicated using a bash comment—for example, `# Output 1.2.2`—<u>it refers to the command immediately following it</u>. This means the highlighted output is generated by that specific command. This formatting makes it easy to see exactly which command produces which result, while still allowing you to copy and paste each command (or series of commands) directly into your terminal and execute them by just pressing <kbd>Enter</kbd>. If a single sequence of commands generates multiple outputs, each specific output for command W.X.Y will be denoted by a letter Z, formatted as 'Output W.X.Y.Z'. At the same time, in various instances, the 'Output W.X.Y.Z' string is also used to point out and provide commentary on the specific W.X.Y.Z command it refers to.
    </Note>
    
    <CodeGroup>
    ```bash Command 1.2.2
    rustc hello_world.rs &&
    # Output 1.2.2
    ./hello_world
    ```
    ```text Output 1.2.2
    Hello, world!
    ```
    </CodeGroup>
  </Step>
</Steps>
